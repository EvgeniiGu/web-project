2.	Физическая файловая система. Организация ФС семейства EXT
	В файловой системе EXT (Extended File System) в Linux данные организованы следующим образом:
v	Superblock
- Суперблок находится в начале файловой системы и содержит основную информацию о ФС, такую как размер ФС, количество блоков, размер блока, тип монтирования и другие параметры.
- Суперблок резервирует несколько блоков в начале ФС для хранения метаданных.
v	Inode Table
- Inode (индексный дескриптор) содержит метаданные каждого файла или каталога, кроме их фактического содержимого.
- Каждый файл имеет свой inode, который хранит информацию, такую как права доступа, владельца, времена доступа и модификации, указатели на блоки данных и другие атрибуты.
v	Data Blocks
- Блоки данных содержат фактические данные файлов и каталогов.
- Данные файлов хранятся в блоках данных.
v	Block Group Descriptor Table
- Таблица дескрипторов групп блоков содержит информацию о каждой группе блоков в ФС.
- Она хранит данные о количестве свободных блоков, inode, битовую карту блоков и другие параметры для каждой группы блоков.
v	Journal
- В журналирующих файловых системах (например, ext3 и ext4) существует журнал, который записывает изменения перед их применением к реальной файловой системе.
